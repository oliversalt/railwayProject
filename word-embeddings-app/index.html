How It Works

In modern Large Language models (LLMs), words are represented as high-dimensional vectors called word embeddings. These embeddings are just a list of numbers that come from training on large amounts of text data. These vectors capture the meanings and relationships between words based on how they are used in context. Each vector itself is abstract to humans but mathematical operations on these vectors can reveal interesting relationships. You can think of the words existing in 3D space, where similar words are closer together. 3blue1brown has a great video explaining this concept in more detail:

The frontend (HTML/CSS/JavaScript) is hosted on GitHub Pages as static files. When you interact with the app, it sends requests to a backend API running on Google Cloud Run.

The backend is a Python FastAPI application running in a Docker container. It uses the GloVe (Global Vectors for Word Representation) model, which has learned word meanings from billions of words of text. The model converts each word into a 50-dimensional vector, and similar words have vectors that are close together in this space.
